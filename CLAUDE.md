# Development Guidelines

## Architecture & Code Quality
- Use reusable components as much as possible to keep pages clean, minimal, and maintainable.
- Keep business logic separated from UI components.
- Follow scalable and modular architecture principles.
- Use clean folder structure and best practices for maintainability.
- Prefer server actions / route handlers where appropriate.

## UI & Styling
- Use shadcn/ui + Tailwind CSS for all UI components and styling.
- Use Framer Motion only when animations or transitions improve the UX.

## Implementation Standards
- This is a full-stack Next.js project — generate complete production-ready implementations including:
  - Database schema
  - Backend logic
  - API routes
  - Authentication (if needed)
  - Validation
  - Error handling
  - Frontend integration
  - State management
  - Real data fetching
- Do NOT use mock data, fake APIs, placeholders, or dummy implementations.
- Do not skip validation, security, or error handling.
- Avoid incomplete examples or partial implementations.
- Generate production-ready code only.
